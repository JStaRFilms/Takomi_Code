# [FR-018] Recovery Workflows

## Labels
`MUS`, `recovery`, `orchestration`

## User Story
As a builder, I want continue-build, migration, reset, and escalation flows so that long-running projects remain recoverable.

## Proposed Solution

### Overview
Map Takomi recovery workflows into explicit session actions with persisted summaries and handoff artifacts.

### Implementation Flow
1. Support continue-build from saved project state.
2. Add migration and reset flows for stuck sessions.
3. Create escalation summaries for handoff or retry.

### Key Considerations
- Recovery actions should preserve the audit trail.
- Handoff artifacts must be easy to inspect and reuse.

## Acceptance Criteria

- [ ] Sessions can resume through a continue-build path.
- [ ] Reset, migration, and escalation actions generate usable handoff artifacts.
- [ ] Recovery workflows remain visible in the session history.
