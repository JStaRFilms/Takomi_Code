# [FR-007] Background Child Execution

## Labels
`MUS`, `orchestration`, `runtime`

## User Story
As a builder, I want child runs to continue in the background so that I can keep working in the parent session.

## Proposed Solution

### Overview
Run child sessions asynchronously while the parent session remains interactive and status-aware.

### Implementation Flow
1. Add background run scheduling.
2. Stream state changes to the parent view.
3. Preserve background run events in the audit log.

### Key Considerations
- Background tasks must be cancellable.
- The parent view must stay responsive during child execution.

## Acceptance Criteria

- [ ] Child runs can continue while the parent session remains usable.
- [ ] Parent sessions receive live state updates from child runs.
- [ ] Background run lifecycle events are persisted.
