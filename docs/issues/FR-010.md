# [FR-010] WinUI Shell and Navigation

## Labels
`MUS`, `desktop-shell`, `ux`

## User Story
As a builder, I want a beautiful native shell with project navigation, session tabs, and inspectors so that the workflow feels clear and fast.

## Proposed Solution

### Overview
Build a WinUI 3 shell with a project rail, session tabs, inspector panels, and Windows-native interaction patterns.

### Implementation Flow
1. Define shell layout and navigation primitives.
2. Create session work area and inspector regions.
3. Bind runtime, branch, and workspace state into the header and side panels.

### Key Considerations
- The app should not feel like a web dashboard in a desktop frame.
- Information density must remain readable on desktop screens.

## Acceptance Criteria

- [ ] Users can navigate projects, sessions, and inspectors without losing context.
- [ ] The shell exposes branch, workspace, and runtime state in primary UI surfaces.
- [ ] The shell is visually aligned with approved design prototypes.
