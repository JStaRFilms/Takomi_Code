# [FR-021] Usage Analytics Dashboard

## Labels
`Future`, `analytics`

## User Story
As a builder, I want app and run analytics so that I can understand throughput, costs, and agent effectiveness.

## Proposed Solution

### Overview
Add reporting surfaces for run outcomes, activity, cost, and intervention frequency once the operational core is stable.

## Acceptance Criteria

- [ ] Core metrics for runs, interventions, and runtimes are defined.
- [ ] Reporting surfaces can query persisted event data.
- [ ] Analytics remain optional and do not block core workflows.
