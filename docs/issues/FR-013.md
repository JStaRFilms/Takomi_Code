# [FR-013] Audit Event Log

## Labels
`MUS`, `audit`, `persistence`

## User Story
As a builder, I want every run, intervention, entitlement change, and verification change logged so that execution is reviewable and trustworthy.

## Proposed Solution

### Overview
Create an append-only event store for operational, billing, and verification events with session and project linkage.

### Implementation Flow
1. Define event categories and payload shapes.
2. Persist events as they occur.
3. Provide UI inspection for event history.

### Key Considerations
- Events must be queryable by project, session, and runtime.
- Sensitive data should be logged safely.

## Acceptance Criteria

- [ ] Runs and interventions create audit events.
- [ ] Billing and verification changes create audit events.
- [ ] Users can inspect event history from the UI.
