# [FR-009] Codex Adapter Runtime

## Labels
`MUS`, `runtime`, `codex`

## User Story
As a builder, I want Takomi to run Codex CLI through a controlled adapter so that code editing remains delegated while the app owns orchestration.

## Proposed Solution

### Overview
Implement a runtime adapter that launches Codex CLI, handles I/O, tracks run state, and supports Windows-specific fallback behavior.

### Implementation Flow
1. Define a runtime interface for local and cloud execution.
2. Implement a Codex CLI adapter for local execution.
3. Capture stdout, stderr, exit state, and event metadata.

### Key Considerations
- Runtime failures must not corrupt session state.
- Windows-specific execution quirks should be isolated in the adapter layer.

## Acceptance Criteria

- [ ] Takomi can start and monitor a Codex-backed local run.
- [ ] Runtime lifecycle events are surfaced in the UI and audit log.
- [ ] Windows fallback handling is isolated from UI logic.
