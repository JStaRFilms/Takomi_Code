# [FR-008] Live Intervention Controls

## Labels
`MUS`, `orchestration`, `controls`

## User Story
As a builder, I want to inject guidance, pause, resume, reroute, replace, cancel, and migrate context so that I stay in control of AI execution.

## Proposed Solution

### Overview
Expose intervention commands as first-class actions on active runs, backed by explicit event records and runtime hooks.

### Implementation Flow
1. Define intervention command types.
2. Wire commands into active run management.
3. Record interventions in the audit log and run transcript.

### Key Considerations
- Intervention history must remain inspectable.
- Unsupported actions must fail safely and visibly.

## Acceptance Criteria

- [ ] Active runs support inject, pause, resume, cancel, reroute, replace, and migrate actions.
- [ ] Intervention commands are visible in the UI and attached to the correct run.
- [ ] Every intervention creates an audit event.
