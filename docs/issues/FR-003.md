# [FR-003] Sub-Chat Hierarchy

## Labels
`MUS`, `orchestration`, `sessions`

## User Story
As a builder, I want child sessions under a parent chat so that orchestration chains stay organized.

## Proposed Solution

### Overview
Model sessions as a tree so parent orchestrator runs can spawn child sessions with inherited context.

### Implementation Flow
1. Add parent-child session edges.
2. Render hierarchy in the UI.
3. Persist inheritance metadata for workspace and runtime context.

### Key Considerations
- Children should inherit defaults without duplicating unrelated state.
- Session tree rendering must remain readable with deep nesting.

## Acceptance Criteria

- [ ] Sessions can create child sessions with parent references.
- [ ] The UI shows parent-child relationships clearly.
- [ ] Child sessions inherit workspace context unless explicitly changed.
