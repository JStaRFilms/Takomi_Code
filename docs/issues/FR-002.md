# [FR-002] Multi-Chat Session System

## Labels
`MUS`, `desktop-shell`, `sessions`

## User Story
As a builder, I want multiple chats per project so that I can separate strategy, build, review, and debugging work.

## Proposed Solution

### Overview
Implement project-scoped chat tabs backed by persistent session records and runtime targets.

### Implementation Flow
1. Create session entities and tab state.
2. Persist transcript metadata and session summaries.
3. Restore open sessions when the app relaunches.

### Key Considerations
- Sessions need stable IDs and project scoping.
- Session tabs must support runtime status badges.

## Acceptance Criteria

- [ ] Users can create multiple chats inside a project.
- [ ] Each chat has its own transcript, metadata, and runtime status.
- [ ] Open chats can be restored after restart.
