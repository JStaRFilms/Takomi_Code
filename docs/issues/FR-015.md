# [FR-015] Paystack Billing and Entitlements

## Labels
`MUS`, `billing`, `payments`

## User Story
As a builder, I want fiat billing and entitlements so that paid access can be enforced.

## Proposed Solution

### Overview
Integrate Paystack for subscription or purchase success paths and map completed payments to entitlement records in the app.

### Implementation Flow
1. Add billing configuration and checkout initiation.
2. Record payment success and entitlement activation.
3. Expose billing state in the shell.

### Key Considerations
- Success-path implementation must be functional, not mocked.
- Billing state changes must be auditable.

## Acceptance Criteria

- [ ] A successful Paystack payment can activate an entitlement.
- [ ] Entitlement checks are available to the application layer.
- [ ] Billing events appear in the audit log.
