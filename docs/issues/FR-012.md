# [FR-012] Git and Worktree Controls

## Labels
`MUS`, `git`, `workspace`

## User Story
As a builder, I want branch visibility and explicit worktree switching so that parallel work stays manageable.

## Proposed Solution

### Overview
Integrate git state into the shell and add an explicit workflow for creating and switching worktrees from any session.

### Implementation Flow
1. Surface branch and dirty state in the UI.
2. Implement create/switch worktree flows.
3. Update session workspace inheritance when a switch occurs.

### Key Considerations
- Same workspace remains the default.
- Worktree switching must be explicit and scoped correctly.

## Acceptance Criteria

- [x] Current branch and dirty state are visible in the shell.
- [x] Users can create and switch worktrees from the app.
- [x] Workspace changes update the correct session subtree.
