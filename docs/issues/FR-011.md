# [FR-011] Transcript and Artifact Persistence

## Labels
`MUS`, `persistence`, `sessions`

## User Story
As a builder, I want chats, events, and generated artifacts saved locally so that sessions can be resumed reliably.

## Proposed Solution

### Overview
Persist transcript turns, session metadata, and artifact references in local storage with workspace-aware file links.

### Implementation Flow
1. Define persistence models for transcripts and artifacts.
2. Store records in SQLite.
3. Rehydrate session state when the app starts.

### Key Considerations
- Artifact links must remain valid across app restarts.
- Session restore should preserve hierarchy and active workspace references.

## Acceptance Criteria

- [ ] Transcript history persists across restarts.
- [ ] Artifact records preserve file references and producing runs.
- [ ] Session restoration rebuilds the previous session hierarchy.
