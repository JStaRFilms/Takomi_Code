# [FR-014] HTML-First Design Review Flow

## Labels
`MUS`, `design`, `ux`

## User Story
As a builder, I want reviewable HTML prototypes before WinUI implementation so that the UX can be refined before engineering hardens it.

## Proposed Solution

### Overview
Use HTML prototypes and a design review loop to validate information architecture, interaction design, and visual hierarchy before build.

### Implementation Flow
1. Produce HTML prototypes for core surfaces.
2. Review prototypes and record changes.
3. Require approval before build work starts.

### Key Considerations
- Prototype coverage must include the key orchestration surfaces.
- Design revisions should update documented source-of-truth artifacts.

## Acceptance Criteria

- [ ] HTML prototypes exist for the required shell surfaces.
- [ ] Design review produces explicit refinement notes.
- [ ] Build work does not begin before mockup approval.
