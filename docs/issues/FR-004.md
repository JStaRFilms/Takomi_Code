# [FR-004] Takomi Mode Loader

## Labels
`MUS`, `takomi`, `config`

## User Story
As a builder, I want the app to read mode YAML files so that orchestrator behavior stays aligned with the Takomi contracts.

## Proposed Solution

### Overview
Load and normalize `.agent/Takomi-Agents/*.yaml` at runtime into internal mode definitions.

### Implementation Flow
1. Read mode files from the workspace.
2. Parse YAML into normalized records.
3. Expose loaded modes to the orchestrator and UI.

### Key Considerations
- Parsing failures must surface actionable errors.
- Missing files should degrade gracefully.

## Acceptance Criteria

- [ ] Mode YAML files are discovered from the workspace.
- [ ] Parsed modes are available to orchestration logic.
- [ ] Invalid mode files generate visible diagnostics.
