# [FR-001] Project and Workspace Management

## Labels
`MUS`, `desktop-shell`, `workspace`

## User Story
As a builder, I want to open and manage multiple project directories so that I can work across products without losing context.

## Proposed Solution

### Overview
Create a project registry with recent projects, explicit workspace context, and durable metadata for runtime, branch, and verification state.

### Implementation Flow
1. Add project import and recent-project persistence.
2. Store active workspace metadata per project.
3. Expose project switching in the shell.

### Key Considerations
- Workspace state must survive app restarts.
- Switching projects must not corrupt active session state.

## Acceptance Criteria

- [ ] Users can add, reopen, and remove project directories.
- [ ] The active workspace context is persisted per project.
- [ ] Project switching updates visible project-specific state in the shell.
